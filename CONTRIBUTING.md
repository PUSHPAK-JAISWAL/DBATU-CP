# Contributing Guidelines

Thank you for considering contributing to this project! Here are some guidelines to help you get started:

## How to Contribute
1. **Fork the repository** - Click on the Fork button on the top right of the repository page.
2. **Clone your fork** - Use the command `git clone https://github.com/your-username/DBATU-CP.git`, replacing `your-username` with your GitHub username.
3. **Create a branch** - Create a new branch for your changes, using `git checkout -b your-branch-name`.
4. **Make your changes** - Implement your changes or features.
5. **Commit your changes** - Use meaningful commit messages to explain your changes. For example: `git commit -m "Add new feature"`
6. **Push your changes** - Push your branch to GitHub with `git push origin your-branch-name`.
7. **Create a Pull Request** - Navigate to the original repository and click on the "New Pull Request" button.

## Code Style
- Follow existing code style and formatting conventions.
- Ensure code is well documented and styled consistently.

## Issues
- Before creating a new issue, check if there’s an existing one.
- Provide clear and concise information about the issue and how to reproduce it.

## Pull Request Guidelines
- Describe your changes clearly in the pull request description.
- Reference related issues if applicable (e.g., `Fixes #123`).
- Be prepared to make changes to your pull request based on feedback.

## License
By contributing, you agree that your contributions will be licensed under the project’s license.

## Thank You!
Thank you for your contributions! Every bit helps and is greatly appreciated!
